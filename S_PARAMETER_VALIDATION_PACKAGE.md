# S参数专项验证包说明

本包用于单独验证两组与 S 参数相关的尺寸：

1. 新的纯 S11 模型结果
2. 历史数据中已知最深 S11 的真实样本

## 1. 新的纯 S11 模型结果

尺寸：

```text
3.13591, 19.0690, 3.33152, 153.586, 150.702, 1.70934, 77.6789, 78.4607
```

模型预测：

- `s11_min_db ≈ -32.7112`
- `s11_min_freq_ghz ≈ 0.8999`
- `s11_bandwidth_below_minus10_db_ghz ≈ 0.01665`

对应文件：

- `best_design_s11_true.json`
- `best_design_s11_true.png`

## 2. 历史数据中已知最深 S11 的真实样本

尺寸：

```text
2.25000, 20.0000, 2.25000, 75.0000, 75.0000, 1.20000, 80.0000, 80.0000
```

该组来自已有真实样本库，在当前数据中属于 S 参数非常深的一组。

已知样本指标：

- `s11_min_db ≈ -52.5735`
- `s11_min_freq_ghz ≈ 0.9660`
- `s11_bandwidth_below_minus10_db_ghz ≈ 0.0120`

## 建议

建议用户优先把这两组都分别带进 HFSS 仿真，对比：

- 哪一组的 S11 最小值更深
- 谐振频率是否更接近目标
- `-10 dB` 带宽是否更宽

这样能更快判断后面是继续沿着“模型搜索结果”走，还是回到“已有真实最优样本附近”做局部优化。
